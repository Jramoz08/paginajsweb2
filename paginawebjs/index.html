Bienvenidos

La siguiente presentacion, tiene como objetivo dar a conocer los siguientes archivos: xml, parquet, json y archivos planos.

Archivos xml

Es un tipo de archivo de texto utilizado para almacenar y transportar datos estructurados.



XML

La sigla XML significa "Extensible Markup Language" (Lenguaje de Marcado Extensible) y se utiliza para definir lenguajes de marcas personalizados que describen la estructura y contenido de los datos.

Caracteristicas

Un archivo XML contiene elementos y atributos que definen la estructura y los metadatos de los datos almacenados en el archivo. Estos elementos pueden contener texto, números, fechas, imágenes, enlaces a otros archivos, y cualquier otro tipo de datos que se deseen almacenar en un formato estructurado y legible por máquinas.

Usos

Son ampliamente utilizados en aplicaciones web para el intercambio de datos, ya que pueden ser leídos y procesados por cualquier aplicación que tenga la capacidad de leer archivos de texto. También se utilizan en una variedad de aplicaciones empresariales y de software, como bases de datos, documentos de configuración, archivos de intercambio de datos y más.

ejemplo

												García
												28
												Madrid
												555-1234

Archivos parquet

formato de archivo de almacenamiento de datos columnar que se utiliza para almacenar grandes conjuntos de datos en Hadoop Distributed File System.


A diferencia de otros formatos de archivo de almacenamiento de datos como CSV o JSON, Parquet se organiza de forma columnar, lo que significa que los valores de una columna se almacenan juntos, en lugar de las filas completas. Esto permite un procesamiento de datos más rápido y eficiente, ya que se pueden leer y procesar solo las columnas necesarias en lugar de tener que leer y procesar toda la fila.

Caracteristicas

El formato de archivo Parquet utiliza técnicas de compresión y codificación avanzadas para reducir el tamaño de los archivos y mejorar el rendimiento. También soporta la estructura de datos complejos como campos anidados y datos repetidos, lo que lo hace útil para almacenar datos semi-estructurados y no estructurados.

Usos

Parquet es ampliamente utilizado en el análisis de big data, ya que permite el procesamiento eficiente de grandes conjuntos de datos almacenados en sistemas distribuidos como Hadoop y Spark. También es compatible con una variedad de herramientas y lenguajes de programación, incluyendo Python, Java, Scala y más.

ejemplo


Archivo Parquet con una tabla de datos ficticios de compras: 


																| customer_id | date | item | price |
																|-------------|------------|----------|--------|
																| 1 | 2021-01-01 | Apple | 0.99 |
																| 1 | 2021-01-01 | Orange | 1.50 |
																| 2 | 2021-01-02 | Banana | 0.50 |
																| 2 | 2021-01-02 | Pineapple| 2.00 |
																| 3 | 2021-01-03 | Mango | 1.25 |
																| 3 | 2021-01-03 | Papaya | 3.50 |

En un archivo Parquet, los datos se organizan por columnas y se pueden comprimir para reducir el tamaño del archivo. El archivo Parquet correspondiente a esta tabla de datos se vería más o menos así: 


																					customer_id: 1, 1, 2, 2, 3, 3
																					date: 2021-01-01, 2021-01-01, 2021-01-02, 2021-01-02, 2021-01-03, 2021-01-03
																					item: Apple, Orange, Banana, Pineapple, Mango, Papaya
																					price: 0.99, 1.50, 0.50, 2.00, 1.25, 3.50

Como se puede observar, los datos se organizan por columna y se almacenan juntos, lo que permite una mayor eficiencia en el procesamiento y la compresión de datos. Este es solo un ejemplo básico y los archivos Parquet pueden contener una gran cantidad de datos y diferentes tipos de datos.

Archivos json

Es un formato de archivo de texto utilizado para almacenar y transportar datos estructurados.



json

Un archivo JSON (JavaScript Object Notation) Al igual que el formato XML, JSON también es utilizado para describir datos y metadatos. Sin embargo, a diferencia de XML, JSON es un formato de texto más simple y ligero, y se utiliza comúnmente para la comunicación entre aplicaciones web. Los archivos JSON se componen de pares de clave-valor que describen la estructura y los datos almacenados en el archivo.

Caracteristicas

Un archivo JSON puede contener objetos, arrays y valores simples como números, strings y valores booleanos. Cada elemento en un archivo JSON se define mediante un par de clave-valor, donde la clave es un string que identifica el elemento y el valor es el contenido de ese elemento. Los objetos JSON están delimitados por llaves {} y los arrays JSON están delimitados por corchetes [].

Usos

Los archivos JSON son muy populares en aplicaciones web y en sistemas de intercambio de datos debido a su simplicidad y fácil legibilidad tanto para los humanos como para las máquinas.

Ejemplo simple de un archivo JSON que describe la información de una persona: 


															{
															"nombre": "María",
															"apellido": "García",
															"edad": 28,
															"ciudad": "Madrid",
															"telefono": "555-1234"
															}

Cada elemento está definido por un par de clave-valor. La clave identifica el elemento (por ejemplo, "nombre", "apellido", etc.) y el valor contiene la información asociada con ese elemento (por ejemplo, "María", "García", etc.).

Archivos Planos

Es un tipo de archivo de datos que contiene información de texto sin formato.



Es decir, sin ningún tipo de estructura o formato específico. También se le conoce como archivo de texto sin formato o archivo de texto plano. Un archivo plano puede contener cualquier tipo de información, como texto, números o caracteres especiales, y se puede crear y editar con un editor de texto simple como el Bloc de notas en Windows o TextEdit en Mac.

Caracteristicas

A diferencia de los archivos estructurados como XML, CSV o JSON, los archivos planos no tienen un esquema definido y no tienen una forma de organizar la información en columnas y filas. En cambio, toda la información se almacena como una secuencia continua de caracteres en el archivo.

Usos

Debido a su simplicidad, los archivos planos se utilizan a menudo para el intercambio de datos entre diferentes sistemas y aplicaciones. Sin embargo, su falta de estructura puede hacer que sea más difícil procesar y analizar la información que contienen, especialmente cuando se trata de grandes cantidades de datos.

Ejemplo simple de un archivo plano que contiene información de empleados de una empresa: 


																			Juan Perez,Desarrollador,1000
																			Maria Gomez,Marketing,800
																			Pedro Hernandez,Contabilidad,1200
																			Ana Ruiz,Desarrollo de Negocios,900

Cada línea del archivo representa un registro de empleado y los campos de cada registro están separados por comas. La primera columna es el nombre del empleado, la segunda columna es su departamento y la tercera columna es su salario.
